Обзор Gemini CLI

Интерактивный справочник по уровням доступа и лимитам использования.

Что такое Gemini CLI?

Gemini CLI (Command-Line Interface) — это инструмент для разработчиков, позволяющий взаимодействовать с моделями искусственного интеллекта Google Gemini прямо из терминала или командной строки. Он идеально подходит для автоматизации задач, интеграции в рабочие процессы разработки и быстрого тестирования возможностей моделей без необходимости создавать графический интерфейс.

2.1. Основные команды и режимы взаимодействия

Gemini CLI предоставляет несколько режимов взаимодействия для оптимизации рабочего процесса. Интерактивный режим (REPL) запускается командой gemini и позволяет вести диалог с агентом, сохраняя контекст беседы. Для выполнения однократных запросов без входа в интерактивную сессию используется неинтерактивный режим с флагом -p (например, gemini -p "Summarize file. @./summary.txt"). Кроме того, инструмент поддерживает пайпинг, что позволяет направлять вывод других команд непосредственно в Gemini CLI (например, echo "Count to 10" | gemini).

Слеш-команды

Внутри интерактивной сессии доступны различные слеш-команды (/) для управления агентом:

  • /help — показывает список доступных команд и их описание.
  • /chat — переключает в режим чата.
  • /memory — позволяет управлять контекстом сессии.
  • /stats — отображает статистику использования, включая количество запросов.
  • /quit — завершает работу инструмента.

2.2. Встроенные инструменты: Ядро агентности

Ключевая возможность Gemini CLI — это использование встроенных инструментов, которые позволяют агенту взаимодействовать с локальным окружением и внешними сервисами. Эти инструменты являются основой архитектуры ReAct и позволяют выполнять задачи, выходящие за рамки простой генерации текста.

Управление файловой системой: Агент может читать (ReadFile, ReadManyFiles) и записывать (WriteFile) файлы, а также находить их по шаблону (FindFiles или glob). Примером практического использования является генерация кода и автоматическое создание новых файлов проекта, таких как app.py или requirements.txt.

Системные команды: Инструмент Shell позволяет выполнять команды оболочки. Это критически важная функция для автоматизации, позволяющая агенту выполнять такие действия, как создание директорий (mkdir), установка пакетов (pip install или npm install) или запуск тестов. Пользователь также может вручную переключиться в режим оболочки, набрав !.

Работа с сетью: Инструменты WebFetch и Google Search позволяют модели получать актуальную информацию из интернета. Это помогает «заземлять» ответы на реальных данных, снижая вероятность галлюцинаций и делая ответы более точными и полезными.

Поскольку некоторые инструменты могут выполнять потенциально деструктивные действия (например, запись файлов или выполнение команд оболочки), в архитектуру Gemini CLI заложен фундаментальный принцип безопасности, известный как «Human in the Loop» (HiTL). Перед выполнением таких операций, как mkdir или WriteFile, агент явно запрашивает у пользователя разрешение. Этот механизм обеспечивает прозрачность и контроль, позволяя разработчику сохранять полный контроль над своим окружением, даже когда задачи автоматизированы. Это критически важный аспект, который укрепляет доверие к инструменту и делает его безопасным для использования в критических средах разработки.

2.3. Управление контекстом и интеграция с IDE

Для эффективной работы в контексте проекта Gemini CLI использует специальные файлы GEMINI.md. Эти файлы содержат инструкции для модели (например, «Весь код на Python должен соответствовать PEP 8» или «Используй отступы в 2 пробела») и могут быть расположены на разных уровнях иерархии проекта. Агент автоматически объединяет контекст из глобального файла (~/.gemini/GEMINI.md) и локальных файлов, находящихся в текущей директории и ее родительских папках, что позволяет точно настроить поведение модели для каждого проекта.

Кроме того, инструмент поддерживает чекпоинтинг, позволяя сохранять и восстанавливать сложные сеансы, а также глубоко интегрируется с популярными IDE, такими как VS Code и Zed. Такая интеграция позволяет агенту получать контекст из редактора, включая открытые файлы и выделенный текст, и визуализировать предлагаемые изменения в виде diff-просмотра, что упрощает их анализ и утверждение.

4. Аутентификация, лимиты и зависимость от тира

Выберите тип лимитов для просмотра информации.

Бесплатный (Flash)

Тип аутентификации: OAuth

Биллинг: Не требуется

Лимит запросов: 60/мин

Лимит токенов: 8 192

Pro

Тип аутентификации: Ключ Gemini API

Биллинг: Бесплатная квота (биллинг не требуется)

Лимит запросов: 1500/мин

Лимит токенов: 32 768

Ultra

Тип аутентификации: Vertex AI

Биллинг: Требуется

Лимит запросов: 3000+/мин

Лимит токенов: 128 000+

Ключевой вывод: зависят ли лимиты от уровня доступа?

Да, и эта зависимость прямая.

OAuth: Бесплатный доступ для индивидуальных разработчиков с базовыми лимитами.

Ключ Gemini API: Бесплатная квота с повышенными лимитами для программного доступа, при этом в рамках бесплатной квоты вам не потребуется биллинг.

Vertex AI: Корпоративное решение с самыми высокими лимитами, где биллинг обязателен.

Обзор методов аутентификации

OAuth-вход

Самый простой способ

Вход через личный аккаунт Google. Не требует управления ключами API и привязан к бесплатной лицензии Gemini Code Assist для частных лиц.

Ключ Gemini API

Для программного доступа

Получается бесплатно в Google AI Studio и устанавливается как переменная окружения GEMINI_API_KEY или в файл .env. Удобен для проектов, где требуется программный доступ без биллинга.

Vertex AI

Корпоративный подход

Интегрирован с Google Cloud Platform. Обеспечивает высокий уровень безопасности, централизованный биллинг и масштабирование, идеально подходит для корпоративных команд и продакшен-нагрузок.
